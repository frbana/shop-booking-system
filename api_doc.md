# 店铺预约系统 API 接口文档

本文档说明店铺预约系统后端的 4 个核心业务接口，适用于本地开发、联调测试和线上部署验证。

## 通用说明

- 基础地址
  - 本地开发：`http://localhost:5000`
  - 线上部署：`https://你的-render-后端域名.onrender.com`
- 统一返回格式

```json
{
  "code": 0,
  "msg": "操作成功",
  "data": {}
}
```

- 返回码说明
  - `code = 0`：成功
  - `code = 1`：失败

---

## 1. 查询时段

### 请求地址

`GET /api/time-slot`

### 请求方式

`GET`

### 入参

无

### 参数说明

无请求参数。接口返回全部预约时段，按日期、开始时间、结束时间升序排列。

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "slot_date": "2026-07-17",
      "start_time": "09:00:00",
      "end_time": "10:00:00",
      "max_capacity": 10,
      "booked_count": 3,
      "remaining_count": 7
    }
  ]
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "查询预约时段失败",
  "data": []
}
```

### 业务限制条件

- 仅返回数据库中已配置的预约时段。
- `remaining_count = max_capacity - booked_count`，不会小于 0。
- 接口为只读接口，不接受修改参数。

## 2. 提交预约

### 请求地址

`POST /api/book-order`

### 请求方式

`POST`

### 入参

请求体 `application/json`

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 预约人姓名。 |
| `phone` | string | 是 | 预约人手机号，必须为 11 位数字。 |
| `slot_id` | number | 是 | 预约时段 ID。 |

### 参数说明

- `name` 不能为空，提交前会去除首尾空格。
- `phone` 必须为 11 位纯数字。
- `slot_id` 必须对应已存在的预约时段。

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "预约成功",
  "data": {
    "booking_id": 12,
    "slot_id": 1,
    "name": "张三",
    "phone": "13800138000",
    "status": "正常",
    "remaining_count": 6
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "手机号必须为11位数字",
  "data": {}
}
```

其他常见失败返回：

```json
{
  "code": 1,
  "msg": "该时段已约满",
  "data": {}
}
```

### 业务限制条件

- 姓名不能为空。
- 手机号必须是 11 位数字。
- 预约时段必须存在。
- 同一手机号在同一时段只能保留 1 笔正常预约。
- 预约满员时禁止继续提交。
- 接口内部通过容量条件更新控制并发超卖风险。

## 3. 活动列表

### 请求地址

`GET /api/activity`

### 请求方式

`GET`

### 入参

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `page` | number | 否 | 页码，默认 `1`。 |
| `size` | number | 否 | 每页条数，默认 `10`，最大 `100`。 |

### 参数说明

- `page` 小于 1 时会自动按 1 处理。
- `size` 小于 1 时会自动按 1 处理，大于 100 时会按 100 处理。
- 返回数据按活动结束时间倒序排列。

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "title": "开业满减",
      "start_at": "2026-07-15 00:00:00",
      "end_at": "2026-07-20 23:59:59",
      "promotion_text": "到店消费立减 20 元",
      "status": "进行中"
    }
  ]
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "查询活动失败",
  "data": []
}
```

### 业务限制条件

- 仅返回数据库中已配置的活动数据。
- 活动状态由当前时间自动计算，分为 `未开始`、`进行中`、`已结束`。
- `page` 和 `size` 只用于分页查询，不影响状态计算。

## 4. 个人预约 / 取消

### 4.1 查询个人预约

#### 请求地址

`GET /api/user/order?phone=手机号`

#### 请求方式

`GET`

#### 入参

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `phone` | string | 是 | 预约人手机号，必须为 11 位数字。 |

#### 参数说明

- 通过手机号查询该手机号对应的全部预约订单。
- 返回结果按创建时间倒序排列。

#### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": [
    {
      "order_id": 12,
      "name": "张三",
      "phone": "13800138000",
      "status": "正常",
      "created_at": "2026-07-17 09:30:00",
      "slot": {
        "slot_id": 1,
        "slot_date": "2026-07-17",
        "start_time": "09:00:00",
        "end_time": "10:00:00"
      }
    }
  ]
}
```

#### 失败返回示例

```json
{
  "code": 1,
  "msg": "手机号必须为11位数字",
  "data": []
}
```

#### 业务限制条件

- `phone` 不能为空。
- `phone` 必须为 11 位数字。
- 只返回与该手机号匹配的预约订单。
- 已取消订单也会被查询出来，状态字段为 `已取消`。

### 4.2 取消预约

#### 请求地址

`PUT /api/user/cancel`

#### 请求方式

`PUT`

#### 入参

请求体 `application/json`

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `order_id` | number | 是 | 订单 ID。接口也兼容查询字符串中的 `order_id`。 |

#### 参数说明

- 支持从请求体读取 `order_id`。
- 若请求体未提供，也会尝试从查询字符串读取 `order_id`。
- 仅能取消状态为 `正常` 的订单。

#### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "取消成功",
  "data": {
    "order_id": 12,
    "status": "已取消",
    "slot_id": 1,
    "remaining_count": 7
  }
}
```

#### 失败返回示例

```json
{
  "code": 1,
  "msg": "仅正常订单可取消",
  "data": {}
}
```

其他常见失败返回：

```json
{
  "code": 1,
  "msg": "订单不存在",
  "data": {}
}
```

#### 业务限制条件

- `order_id` 不能为空，且必须为整数。
- 订单必须存在。
- 仅 `正常` 状态订单可取消。
- 取消后订单状态变为 `已取消`，并释放对应时段名额。
