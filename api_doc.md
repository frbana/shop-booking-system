# 店铺预约系统 API 接口文档

本文档说明店铺预约系统后端接口，适用于本地开发、前后端联调、Postman 测试和线上部署验证。

## 通用说明

- 本地基础地址：`http://127.0.0.1:5001`
- 线上基础地址：`https://你的-render-后端域名.onrender.com`
- 请求体格式：`application/json`
- 统一返回格式：

```json
{
  "code": 0,
  "msg": "操作成功",
  "data": {}
}
```

返回码说明：

| code | 说明 |
| --- | --- |
| `0` | 请求成功 |
| `1` | 请求失败 |

---

## 1. 健康检查

### 请求地址

`GET /api/health`

### 请求方式

`GET`

### 入参

无

### 参数说明

用于检查后端服务是否正常启动。

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "status": "healthy"
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "服务器内部错误",
  "data": {}
}
```

### 业务限制条件

- 不依赖登录状态。
- 不修改数据库数据。

## 2. 查询预约时段

### 请求地址

`GET /api/time-slot`

### 请求方式

`GET`

### 入参

无

### 参数说明

返回全部预约时段，按预约日期、开始时间、结束时间升序排列。

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "slot_date": "2026-07-19",
      "start_time": "09:00:00",
      "end_time": "10:00:00",
      "max_capacity": 6,
      "booked_count": 1,
      "remaining_count": 5
    }
  ]
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "查询预约时段失败",
  "data": []
}
```

### 业务限制条件

- 仅返回数据库中已配置的预约时段。
- `remaining_count = max_capacity - booked_count`，并且不会小于 0。
- 接口为只读接口，不接受修改参数。

## 3. 提交预约

### 请求地址

`POST /api/book-order`

### 请求方式

`POST`

### 入参

```json
{
  "name": "张三",
  "phone": "13800138000",
  "slot_id": 1
}
```

### 参数说明

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 预约人姓名，不能为空。 |
| `phone` | string | 是 | 预约人手机号，必须为 11 位数字。 |
| `slot_id` | number | 是 | 预约时段 ID，必须存在。 |

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "预约成功",
  "data": {
    "booking_id": 12,
    "slot_id": 1,
    "name": "张三",
    "phone": "13800138000",
    "status": "正常",
    "remaining_count": 4
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "该手机号已预约此时段",
  "data": {}
}
```

其他常见失败返回：

```json
{
  "code": 1,
  "msg": "该时段已约满",
  "data": {}
}
```

### 业务限制条件

- 姓名不能为空。
- 手机号必须为 11 位数字。
- 预约时段必须存在。
- 同一手机号在同一时段只能保留 1 笔 `正常` 预约。
- 预约时段满员后禁止继续提交。
- 后端通过条件更新减少并发超卖风险。

---

## 4. 查询活动列表

### 请求地址

`GET /api/activity`

### 请求方式

`GET`

### 入参

| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | number | 否 | `1` | 页码，小于 1 时按 1 处理。 |
| `size` | number | 否 | `10` | 每页条数，最小 1，最大 100。 |

### 参数说明

示例：

```text
GET /api/activity?page=1&size=10
```

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "title": "新客到店体验礼",
      "start_at": "2026-07-17 00:00:00",
      "end_at": "2026-07-28 23:59:59",
      "promotion_text": "新用户预约到店可享 9 折体验优惠",
      "status": "进行中"
    }
  ]
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "查询活动失败",
  "data": []
}
```

### 业务限制条件

- 仅返回数据库中已配置的活动数据。
- 活动状态由后端根据当前时间计算，可能为 `未开始`、`进行中`、`已结束`。
- 返回数据按活动结束时间倒序排列。

## 5. 用户注册

### 请求地址

`POST /api/user/register`

### 请求方式

`POST`

### 入参

```json
{
  "account": "zhangsan",
  "password": "123456",
  "phone": "13800138000",
  "username": "张三"
}
```

### 参数说明

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `account` | string | 是 | 登录账号，不能为空，不能重复。 |
| `password` | string | 是 | 登录密码，至少 6 位。 |
| `phone` | string | 是 | 绑定手机号，必须为 11 位数字，不能重复绑定。 |
| `username` | string | 否 | 用户名；为空时默认使用账号名。 |

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "注册成功",
  "data": {
    "id": 1,
    "account": "zhangsan",
    "phone": "13800138000",
    "username": "张三",
    "gender": "未设置",
    "birthday": "",
    "avatar": ""
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "该手机号已绑定其他账号",
  "data": {}
}
```

其他常见失败返回：

```json
{
  "code": 1,
  "msg": "账号已存在",
  "data": {}
}
```

### 业务限制条件

- 账号不能为空且必须唯一。
- 密码至少 6 位。
- 手机号必须为 11 位数字。
- 每个手机号只能绑定一个账号。
- 密码以哈希形式写入数据库，不直接返回密码字段。

## 6. 用户登录

### 请求方式

`POST`

### 入参

```json
{
```

### 参数说明

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `account` | string | 是 | 登录账号。 |
| `password` | string | 是 | 登录密码。 |

### 成功返回 JSON 示例

```json
{
  "code": 0,

  "data": {
    "id": 1,
    "account": "zhangsan",
    "phone": "13800138000",
```

### 失败返回示例

```json
{
  "code": 1,
```

## 7. 保存用户资料

### 请求地址

`PUT /api/user/profile`

### 请求方式

`PUT`

### 入参

```json
{
  "user_id": 1,
  "username": "张三",
  "gender": "男",
  "birthday": "2000-01-01",
  "avatar": "data:image/png;base64,..."
}
```

### 参数说明

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `user_id` | number | 是 | 用户 ID，必须存在。 |
| `username` | string | 是 | 用户名，不能为空。 |
| `gender` | string | 否 | 性别，可选值：`未设置`、`女`、`男`、`其他`。 |
| `birthday` | string | 否 | 生日，格式为 `YYYY-MM-DD`，为空表示未设置。 |
| `avatar` | string | 否 | 头像 DataURL 字符串，前端本地上传图片后生成。 |

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "保存成功",
  "data": {
    "id": 1,
    "account": "zhangsan",
    "phone": "13800138000",
    "username": "张三",
    "gender": "男",
    "birthday": "2000-01-01",
    "avatar": "data:image/png;base64,..."
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "生日格式必须为YYYY-MM-DD",
  "data": {}
}
```

其他常见失败返回：

```json
{
  "code": 1,
  "msg": "用户不存在",
  "data": {}
}
```

### 业务限制条件

- 用户 ID 必须存在。
- 用户名不能为空。
- 性别只能是 `未设置`、`女`、`男`、`其他`。
- 生日必须是 `YYYY-MM-DD` 格式或空字符串。
- 头像保存为 DataURL 字符串，适合实训演示；生产环境建议使用对象存储保存图片文件。
- 该接口不允许修改账号、手机号和密码。

## 8. 查询个人预约

### 请求地址

`GET /api/user/order?phone=手机号`

### 请求方式

`GET`

### 入参

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `phone` | string | 是 | 预约手机号，必须为 11 位数字。 |

### 参数说明

个人中心登录后使用当前账号绑定手机号调用该接口。

示例：

```text
GET /api/user/order?phone=13800138000
```

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": [
    {
      "order_id": 12,
      "name": "张三",
      "phone": "13800138000",
      "status": "正常",
      "created_at": "2026-07-18 09:30:00",
      "slot": {
        "slot_id": 1,
        "slot_date": "2026-07-19",
        "start_time": "09:00:00",
        "end_time": "10:00:00"
      }
    }
  ]
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "手机号必须为11位数字",
  "data": []
}
```

### 业务限制条件

- `phone` 不能为空。
- `phone` 必须为 11 位数字。
- 返回与该手机号匹配的所有预约订单。
- 已取消订单也会返回，状态字段为 `已取消`。

## 9. 取消预约

### 请求地址

`PUT /api/user/cancel`

### 请求方式

`PUT`

### 入参

```json
{
  "order_id": 12
}
```

### 参数说明

| 参数名 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `order_id` | number | 是 | 订单 ID。接口也兼容查询字符串中的 `order_id`。 |

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "取消成功",
  "data": {
    "order_id": 12,
    "status": "已取消",
    "slot_id": 1,
    "remaining_count": 6
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "仅正常订单可取消",
  "data": {}
}
```

其他常见失败返回：

```json
{
  "code": 1,
  "msg": "订单不存在",
  "data": {}
}
```

### 业务限制条件

- `order_id` 不能为空，且必须为整数。
- 订单必须存在。
- 仅 `正常` 状态订单可取消。
- 取消后订单状态变为 `已取消`，并释放对应时段名额。

---

## 10. 数据统计

### 请求地址

`GET /api/summary`

### 请求方式

`GET`

### 入参

无

### 参数说明

返回预约时段、预约订单和店铺活动数量。

### 成功返回 JSON 示例

```json
{
  "code": 0,
  "msg": "查询成功",
  "data": {
    "time_slot_count": 150,
    "booking_count": 7,
    "activity_count": 8
  }
}
```

### 失败返回示例

```json
{
  "code": 1,
  "msg": "服务器内部错误",
  "data": {}
}
```

### 业务限制条件

- 只返回数量统计。
- 不返回用户数量和用户敏感信息。
